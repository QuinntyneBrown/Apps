## Design Principles

### Visual Hierarchy
1. **Primary Actions**: Large, prominent buttons (Log Activity, Set Goal)
2. **Secondary Actions**: Smaller buttons (View All, Export)
3. **Tertiary Actions**: Text links (View Details, Edit)

### Color Coding
- **Success/Positive**: Green (#4CAF50)
- **Warning/Moderate**: Yellow/Orange (#FF9800)
- **Error/Severe**: Red (#F44336)
- **Info**: Blue (#2196F3)
- **Neutral**: Gray (#9E9E9E)

### Typography
- **Headings**: Bold, 18-24px
- **Body Text**: Regular, 14-16px
- **Labels**: Semi-bold, 12-14px
- **Captions**: Regular, 10-12px

### Spacing
- **Section Padding**: 16-24px
- **Component Margin**: 12-16px
- **Input Field Padding**: 12px
- **Button Padding**: 12px vertical, 24px horizontal

### Responsive Breakpoints
- **Mobile**: < 768px (single column, bottom nav)
- **Tablet**: 768-1024px (two columns, side nav)
- **Desktop**: > 1024px (multi-column, persistent nav)

### Accessibility
- Minimum touch target: 44x44px
- Color contrast ratio: 4.5:1
- Keyboard navigation support
- Screen reader labels
- Focus indicators

---

## Interaction Patterns

### Form Validation
- **Inline Validation**: Real-time feedback as user types
- **Field-Level Errors**: Red border + error message below field
- **Form-Level Errors**: Summary at top of form
- **Success States**: Green checkmark + success message

### Loading States
- **Skeleton Screens**: Placeholder content while loading
- **Spinners**: For quick operations (< 3 seconds)
- **Progress Bars**: For longer operations with known duration
- **Pull-to-Refresh**: On mobile list views

### Empty States
- **Informative Messages**: Explain why content is empty
- **Call-to-Action**: Prompt user to create first item
- **Illustrations**: Visual elements to enhance empty state

### Notifications
- **Toast Notifications**: Auto-dismiss after 3-5 seconds
- **Persistent Alerts**: Require user dismissal
- **Badge Counts**: Show unread/pending items
- **Push Notifications**: For mobile apps
