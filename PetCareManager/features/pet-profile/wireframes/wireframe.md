## Design Principles

1. **Progressive Disclosure**: Show essential information first, hide complexity
2. **Visual Hierarchy**: Use size, color, spacing to guide attention
3. **Clear CTAs**: Primary actions prominently placed and styled
4. **Feedback**: Immediate feedback for all user actions
5. **Accessibility**: Keyboard navigation, screen reader support, sufficient contrast
6. **Mobile-First**: Design for smallest screen, enhance for larger
7. **Consistency**: Reuse patterns and components across screens
8. **Data Density**: Balance information richness with whitespace
9. **Error Prevention**: Validate early, guide users, confirm destructive actions
10. **Delightful Details**: Micro-interactions, smooth transitions, thoughtful empty states

---

## Responsive Breakpoints

- **Mobile:** < 640px (Single column, stacked layouts)
- **Tablet:** 640px - 1024px (2-column grids, side-by-side)
- **Desktop:** > 1024px (3+ column grids, sidebar layouts)

---

## Interaction Patterns

### Hover States
- Cards: Lift shadow, border highlight
- Buttons: Darken background, show icon
- Links: Underline, color change

### Loading States
- Cards: Skeleton screens with shimmer
- Lists: Progressive loading
- Buttons: Spinner, disabled state

### Error States
- Form fields: Red border, error icon, message below
- Page level: Alert banner at top
- Empty states: Friendly message, helpful action

### Success States
- Toast notifications (4 seconds)
- Inline success messages
- Checkmark animations

---

## Accessibility Features

- **Keyboard Navigation**: Tab order, focus indicators, shortcuts
- **Screen Readers**: ARIA labels, semantic HTML, alt text
- **Color Contrast**: WCAG AA compliance (4.5:1 minimum)
- **Touch Targets**: Minimum 44x44px for mobile
- **Error Handling**: Clear messages, multiple indicators (not color alone)
- **Form Labels**: Properly associated, always visible
